# 工数管理システム（管理用 / クライアント共有用）

プランC（実働時間契約）の稼働管理ツール。1つのアプリ内に「弊社管理ビュー」と「クライアント共有ビュー」を持ち、
`localStorage` でデータを共有・永続化します。

## 起動

```bash
npm install
npm run dev      # http://localhost:5190
npm run build    # 型チェック + 本番ビルド
```

## 実装済み機能

**トップ（プロジェクト選択・管理者専用想定）**
- プロジェクトの追加／選択／削除。案件ごとに独立管理（今回以外でも使用可）。

**弊社管理ビュー**
- 担当（私／外注コーダー、追加可）を選び、作業を **開始／中断／終了** で計測（複数同時計測可・ライブ表示）。
- 押し忘れ対策：アーカイブから各記録の時刻を **手動編集**、または **手動で記録を追加**。
- 計測対象：デザイン ／ コーディング作業（難易度A）／（難易度B）。**作業種の追加・削除**（コーディングC/D等）に対応。計測済みは削除不可。
- **単価はコーディングのみ**設定（外注コスト管理用）。**一度確定すると変更不可**。デザインは単価対象外。
- 月次サマリー：作業別の時間と **外注原価**（コーディングの単価×時間）。
- 月ごとのアーカイブ（いつ作業したか）。
- **作業予定**：今月の予定を登録し、完了チェックで「完了」表示。

**クライアント共有ビュー（閲覧専用）**
- 管理データを読み取り表示。開始/中断/終了/編集ボタンなし。
- **現在作業中**をライブ表示（誰がどの作業をしているか）。
- 月次で3作業の時間と **費用（時給7,000円）**。
- 月ごとのアーカイブ、作業予定の閲覧。

## 構成
```
src/
  App.tsx                 プロジェクト選択・ビュー切替・月ナビ
  store.ts                localStorage ストア＋全アクション、useNow（ライブ計測）
  types.ts / util.ts      型 / 時間・金額・月ユーティリティ（CLIENT_RATE=7000）
  components/
    ProjectHome.tsx       トップ（プロジェクト一覧・作成）
    AdminView.tsx         弊社管理ビュー
    ClientView.tsx        クライアント共有ビュー
    WorkTag.tsx           作業種タグ
```

## ⚠️ 本番共有について（重要）

現状は **localStorage 版**です。同一ブラウザ内では「管理」「共有」両ビューが同じデータを共有し、
別タブ間も同期しますが、**別端末のクライアントがあなたの入力をリアルタイムに見るには、共有バックエンドが必要**です。
本番運用では **Firebase Firestore 等**に置き換え、クライアントには共有ビューのURLのみを配布する構成を推奨します
（全業務ロジックは本アプリのまま流用可能）。
